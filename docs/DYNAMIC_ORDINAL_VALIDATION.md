# Dynamic ordinal numerical validation

The maintained dynamic random-effects Ordered Logit numerical design contains
60 entities observed for six internally consecutive periods. It is generated by
`simulate_dynamic_random_effects_ordered_logit` from the exact conditional model
implemented by the estimator.

## Certification checks

### Quadrature convergence

A 12-point Gaussian-Hermite fit is compared with a 20-point fit under identical
data and model controls. Maintained tolerances are:

- log-likelihood absolute difference: `1e-6`;
- maximum parameter absolute difference: `2e-5`;
- maximum fitted-probability absolute difference: `5e-6`.

The calibration run underlying these guards produced differences of roughly
`1.0e-8`, `1.2e-5`, and `3.6e-6`, respectively.

### Data-representation invariance

The same fitted model is checked after:

- jointly permuting all observation rows;
- replacing numeric entities with arbitrary permuted string labels;
- shifting the time origin by 2,000 without changing time differences.

Row-order and time-origin comparisons use a `2e-6` parameter tolerance. Entity
relabeling uses `1e-5` because changing group summation order can introduce small
floating-point differences.

These checks certify numerical stability for the maintained design and option
set. They are not a universal equivalence claim over every panel structure,
quadrature rule, or initial-conditions specification.
